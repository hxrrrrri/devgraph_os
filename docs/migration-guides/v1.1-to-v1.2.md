# Migrating from v1.1 to v1.2

v1.2 is additive — no schema migration is required and v1.1 graphs continue
to load. New metadata fields default to empty.

## What changed in the schema

### Node `metadata` additions
- `metadata.framework` is now set on every `api_endpoint` and on classes
  matching a framework signature (Spring, Rails, Laravel, SQLAlchemy,
  Prisma).
- Component classes / functions in React files carry
  `metadata.kind = "component"` and an optional `metadata.react_hooks` list.
- Alembic operations emit `schema` nodes with `metadata.operation`,
  `metadata.target`, and `metadata.framework = "alembic"`.
- Prisma models emit `schema` + `database_table` synonyms with
  `metadata.framework = "prisma"`.

### `ReviewResult` additions
The following fields were added with `default_factory=list/dict`. Old review
JSON dumps loaded by v1.2 will simply have empty defaults:
- `api_signature_changes`
- `route_contract_changes`
- `fan_out`
- `infra_blast_radius`
- `severity_by_file`
- `severity_by_symbol`
- `migration_warnings` was already present but is now consistently populated.

### TS types
`@devgraph/schema` `reviewResultSchema` mirrors the Python additions. The
Zod schema falls back to `[]` / `{}` for missing fields, so dashboards built
against v1.1 still load v1.2 payloads.

## Files / configuration

- `.prisma` is now recognized as code. Add to `.devgraphignore` if you
  intentionally don't want it indexed.
- VS Code views: `devgraph.changedView`, `devgraph.impactedView`,
  `devgraph.riskyView`. They appear automatically; no user action needed.

## Behavioral changes

- Express-style routes now carry `framework="express"`. Code that read
  `metadata.framework` and assumed it was always missing on Express routes
  must update its expectations.
- React function components are now tagged `framework="react"`. If you have
  downstream code that relied on absence of `framework`, treat React as a
  valid framework label.

## Recommended steps after upgrade

1. Run `devgraph build --force` to re-extract files with the new plugins.
2. (Optional) Create a snapshot with `devgraph snapshot baseline` so future
   reviews can diff against it for `api_signature_changes` and
   `route_contract_changes`. (Snapshot file lives under
   `.devgraph/snapshots/<name>.json`.)
3. Open the dashboard — the Review lens shows the new severity heat map and
   compat warnings; the Graph lens shows the new framework metadata.

## No breaking changes

- The CLI surface is unchanged.
- The MCP tool surface is unchanged.
- The graph storage SQLite schema is unchanged (new metadata lives in the
  existing JSON `metadata` columns).
